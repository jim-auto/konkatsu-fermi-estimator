# 恋人条件シミュレーター

<p align="center">
  <img src="./docs/readme-preview.svg" alt="恋人条件シミュレーター preview" width="100%">
</p>

恋人条件を重ねると、候補がどれくらい残るかをフェルミ推定で可視化するツールです。

## デモ説明

検索対象の性別、年齢、居住地、未婚、IQ、家事スキル、運動神経、年収、学歴、身長、外見偏差値、スペ値、カップ数、経験人数、夜職経験、ハイブラ興味、上京経験などの条件を ON/OFF できます。条件を追加するたびに、母集団に対して仮定した割合を掛け合わせ、残存人数がグラフ上で減っていきます。

初期状態は「極端: 女性」のプリセットです。10代後半〜20代前半女性・都市部・未婚・IQ115以上・家事かなり得意・運動神経かなり得意・外見偏差値65以上・スペ値105以上・Eカップ・経験人数0人・夜職経験なし・ハイブラ興味なし・上京経験なしで、母集団がどれくらい残るかをすぐ確認できます。

## 例

「IQ115以上・家事かなり得意・運動神経かなり得意・顔面偏差値65以上・スペ値105以上・大卒以上・年収800万以上・身長175cm以上・10代後半〜20代前半・都市部・未婚」で絞ると、まず対象年齢の男性人口に減り、そこから未婚率、都市居住、IQ、家事スキル、運動神経、外見、体型、学歴、年収、身長条件が順番に掛け合わされます。各条件は単体では現実的に見えても、複数を同時に満たす人数は急速に少なくなります。

## 注意書き

- このツールは推定です。
- 公的統計を参考にした近似と、エンタメ寄りの仮定が混在しています。
- 実際の人口、未婚率、所得分布、居住地分布、個人の可能性とは異なる可能性があります。
- 条件同士の相関は単純化しており、原則として独立に近いものとして乗算しています。
- IQ、家事スキル、運動神経、外見偏差値、スペ値、学歴、カップ数、経験人数、夜職経験、ハイブラ興味、上京経験は特に仮定が強い条件です。
- 数値の目的は統計的な正確さではなく、「条件を増やすと母集団が急減する」感覚を掴むことです。
- 係数ごとの確認状況は [docs/fact-check.md](./docs/fact-check.md) にまとめています。

## 機能

- React + TypeScript + Vite で実装
- 条件 ON/OFF とセレクト操作によるリアルタイム更新
- 男女別に異なる分布を利用
- 年齢・IQ・身長・経験人数は最小/最大レンジで範囲指定
- 共通条件として IQ、家事スキル、運動神経を指定可能
- 女性検索時のカップ数（Hカップ以上まで）、経験人数レンジ、夜職経験、ハイブラ興味、上京経験条件
- 男性検索時の顔面偏差値、スペ値、学歴、年収、身長レンジ条件
- 条件ごとの残存人数をログ幅バーとステップカードで可視化
- 平均条件との比較
- 男女比較モード
- 初期状態で極端条件プリセットを適用
- 年齢人口、未婚率、年収、身長は公的統計に寄せた係数を利用
- `src/data` と `src/utils` を分離し、将来的な API 化を想定

## GitHub Pages で公開

このリポジトリは GitHub Pages の project site として公開できます。

1. GitHub に push します。
2. Repository の `Settings` -> `Pages` を開きます。
3. `Build and deployment` の `Source` を `GitHub Actions` にします。
4. `main` ブランチに push すると `.github/workflows/deploy.yml` が `dist/` を自動デプロイします。

公開 URL は通常、次の形式になります。

```text
https://<github-user-name>.github.io/konkatsu-fermi-estimator/
```

GitHub Pages 用ビルドでは Vite の `base` が `/konkatsu-fermi-estimator/` へ切り替わります。ローカルで同じビルドを確認する場合は次を使います。

```bash
npm run build:pages
```
